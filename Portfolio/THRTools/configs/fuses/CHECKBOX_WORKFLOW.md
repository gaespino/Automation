# Fuse File Generator - Checkbox-Based Workflow

## Overview
The Fuse File Generator now uses a **checkbox-based approach** for multi-socket and multi-IP configuration. This makes it crystal clear what combinations will be generated before assigning values.

## New Workflow

### Step 1: Select Sockets
Choose which sockets to target using checkboxes:
- **All Sockets (plural)** - Applies to all sockets
- **Socket 0** - Specific socket
- **Socket 1** - Specific socket

You can select **multiple sockets** simultaneously.

### Step 2: Select IP Instances
Choose which IP instances to target using product-specific checkboxes:

**For GNR/CWF Products:**
- **Computes Section:**
  - All Computes (plural)
  - compute0
  - compute1
  - compute2

- **IOs Section:**
  - All IOs (plural)
  - io0
  - io1

**For DMR Products:**
- **CBBs Section:**
  - All CBBs (plural)
  - cbb0
  - cbb1
  - cbb2
  - cbb3

- **IMHs Section:**
  - All IMHs (plural)
  - imh0
  - imh1

You can select **multiple IPs** from different sections simultaneously.

### Step 3: Generate Combination List
Click the **"⚡ Generate Combination List"** button to create all socket × IP × fuse combinations.

**Example:**
If you have:
- 2 fuses selected
- Sockets: socket0, socket1 (2 selected)
- IPs: compute0, compute1 (2 selected)

The result will be: **2 × 2 × 2 = 8 rows** in the table:
1. socket0 | compute0 | fuse1 | value
2. socket0 | compute0 | fuse2 | value
3. socket0 | compute1 | fuse1 | value
4. socket0 | compute1 | fuse2 | value
5. socket1 | compute0 | fuse1 | value
6. socket1 | compute0 | fuse2 | value
7. socket1 | compute1 | fuse1 | value
8. socket1 | compute1 | fuse2 | value

### Step 4: Assign Values
The table shows all combinations with columns:
- **Socket** - Target socket (sockets, socket0, socket1)
- **IP Instance** - Target IP instance (compute0, computes, io1, etc.)
- **Fuse Instance** - The fuse register name
- **Fuse Width** - Bit width of the fuse
- **Description** - Human-readable description
- **Value** - Assigned hex value (editable)

**Editing Values:**
- **Double-click** the Value column to edit individual values
- Use **Quick Apply** toolbar to batch-assign values:
  - "Apply to All" - Sets all rows to the same value
  - "Apply to Selected Rows" - Sets selected rows to the same value

### Step 5: Generate Fuse File
Click **"✓ Generate Fuse File"** to create the .fuse configuration file.

The file will be organized into sections based on socket and IP combinations:

```ini
# Fuse configuration file for GNR
# Generated by PPV Engineering Tools - Fuse File Generator
# Total combinations: 8
# Unique fuses: 2

[sv.socket0.compute0.fuses]
register_name1 = 0x1
register_name2 = 0xFF

[sv.socket0.compute1.fuses]
register_name1 = 0x1
register_name2 = 0xFF

[sv.socket1.compute0.fuses]
register_name1 = 0x2
register_name2 = 0xAA

[sv.socket1.compute1.fuses]
register_name1 = 0x2
register_name2 = 0xAA
```

## Benefits of Checkbox Approach

### 1. **Clarity**
- You see exactly what combinations will be created **before** assigning values
- No hidden per-row dropdowns - everything is explicit

### 2. **Flexibility**
- Select multiple sockets and IPs at once
- Easily create configurations for all combinations of socket+IP

### 3. **Efficiency**
- Generate all combinations with one click
- Batch value assignment with Quick Apply tools
- Excel-like table for easy editing

### 4. **Validation**
- Shows total combinations before generation
- Validates socket and IP selections
- Clear error messages if selections are missing

## Common Use Cases

### Scenario 1: Test All Computes
**Goal:** Apply same fuse values to all compute instances

**Steps:**
1. Check: socket0, socket1
2. Check: compute0, compute1, compute2
3. Generate List → Creates 2 × 3 × N = 6N rows
4. Use "Apply to All" to set same value for all

### Scenario 2: Different Values Per Socket
**Goal:** socket0 gets 0x1, socket1 gets 0x2

**Steps:**
1. Check: socket0, socket1
2. Check: computes (plural)
3. Generate List → Creates 2 × 1 × N = 2N rows
4. Select all socket0 rows → Apply 0x1
5. Select all socket1 rows → Apply 0x2

### Scenario 3: Specific IP Configuration
**Goal:** Only target compute0 and io0

**Steps:**
1. Check: sockets (plural)
2. Check: compute0, io0
3. Generate List → Creates 1 × 2 × N = 2N rows
4. Assign values individually or use batch apply

## Tips & Tricks

- **Start Simple:** Begin with fewer selections to understand the combinations
- **Use Plural Forms:** Select "computes" or "sockets" to apply to all instances
- **Batch Operations:** Use Quick Apply to speed up value assignment
- **Sort Table:** Click column headers to sort by socket, IP, or fuse name
- **Excel-like:** Alternating row colors make it easy to read large tables
- **Value Editing:** Double-click any value to open detailed editor dialog

## Keyboard Shortcuts

- **Double-Click Value Column** - Edit value
- **Ctrl+Click** - Multi-select rows
- **Shift+Click** - Range select rows
- **Enter in Value Editor** - Apply and close

## Validation

The tool validates:
- ✓ At least one socket must be selected
- ✓ At least one IP must be selected
- ✓ Hex values must be valid (0x prefix, hex digits, or binary 0b prefix)
- ✓ All combinations shown before file generation

## Generated File Format

Section headers follow the format:
```
[sv.{socket}.{ip}.fuses]
```

Where:
- `{socket}` = sockets | socket0 | socket1
- `{ip}` = computes | compute0 | compute1 | compute2 | ios | io0 | io1 | cbbs | cbb0-3 | imhs | imh0-1

All fuse instance names are automatically converted to **lowercase** in the generated file.
